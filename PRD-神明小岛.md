# 《神明小岛 / Deus Isle》产品需求文档（PRD v1.0）

> 一款基于 Web 的 Low-Poly 三渲二 3D 模拟经营游戏：玩家扮演上帝，从远古部落起步，引导一座小岛城市跨越八大文明纪元，直至太空时代。

---

## 1. 一句话概念

**"你是这座岛的上帝——但你只能引导，不能代劳。"**

玩家以"神"的视角俯瞰一座漂浮小岛，通过施展神迹、规划土地、引导科技，让岛上的文明从钻木取火一路进化到曲率引擎。岛民的每一个时代，都留下肉眼可见的层叠历史：金字塔旁可能立着民国工厂，工厂旁边是量子塔——**这座岛本身就是一部可以走进去的文明史**。

---

## 2. 核心设计目标（怎么好玩）

### 2.1 三个支柱

| 支柱 | 说明 |
|---|---|
| **看得见的进化** | 每一次时代跃迁，整座岛的视觉风格、音乐、市民外观、交通工具全部"换皮重生"。升级时代是全游戏最强的多巴胺时刻，要做成一场"文明蜕变秀"。 |
| **神性的克制** | 玩家是神，但不是全能作弊器。神力受"信仰值"限制，强行干预（如天降资源）会让市民产生依赖或恐惧，长期反而不利。好玩之处在于**引导与放手的张力**。 |
| **一岛一史** | 岛面积有限，扩张靠"填海"与"浮岛"。土地是最稀缺资源，每个时代的建筑占地、效率、美学都不同，逼玩家做取舍：拆了古罗马斗兽场盖发电厂吗？历史建筑有"遗产加成"，让取舍真正纠结。 |

### 2.2 创意亮点（差异化设计）

1. **时代叠层（Era Strata）**：旧建筑不会被强制拆除，而是成为"遗迹"。遗迹产生旅游业与文化值，但占用宝贵土地。玩家自然形成"博物馆区/工业区/新城"的城市肌理，每座玩家的岛都独一无二。
2. **神迹系统（Miracles）**：消耗信仰值施展——赐雨、地震（有代价）、天降流星矿、启迪某位"伟人"（触发科技飞跃）。神迹有冷却与副作用，例如频繁赐雨会让市民不再修水利。
3. **市民即个体**：重要市民有名字、职业、心情，镜头可以跟拍一个渔民从出生到去世。市民会"祈祷"，屏幕边缘出现祈祷气泡——这是游戏给玩家的任务系统，用叙事包装。
4. **文明岔路**：每次时代跃迁时玩家二选一（如：工业革命选"煤炭巨兽"还是"电气先驱"），影响后续建筑外观、资源链与终局奇观。增加重玩性。
5. **岛的"性格"**：市民 AI 会根据玩家行为形成集体性格——仁慈的神养出懒散但幸福的市民，严苛的神养出高效但压抑的市民。性格影响事件、效率与祈祷内容。
6. **终局奇观**：每个时代可建一座时代奇观（金字塔、万里长城、自由女神级地标、太空电梯）。太空时代的终极奇观=把整座岛发射成一艘"方舟飞船"，通关动画是整个岛脱离海面升空。

## 3. 核心循环（Core Loop）

```
观察岛屿 → 收到祈祷/事件 → 规划与建造 → 市民自动工作/生活
   ↑                                              ↓
   └──── 收获资源/信仰/幸福 ← 时代进度条推进 ←───┘
                        ↓
              满足条件 → 时代跃迁演出 → 新建筑/新科技/新挑战
```

- **30 秒循环**：点选、建造、看市民走动，获得即时反馈（金币弹跳、爱心、烟雾）。
- **5 分钟循环**：完成一次祈祷任务、解锁一栋关键建筑、扩张一块土地。
- **30 分钟循环**：达成时代跃迁条件，观看蜕变演出，进入新纪元。

## 4. 时代系统（Eras）

| # | 时代 | 视觉基调 | 标志性建筑 | 核心新机制 |
|---|---|---|---|---|
| 1 | 远古时期 | 葱郁原始、篝火暖光 | 图腾柱、茅屋、祭坛 | 采集/狩猎、人口诞生、信仰解锁 |
| 2 | 古埃及/罗马 | 沙漠金+大理石白 | 金字塔、神庙、引水渠 | 农业灌溉、奴隶/公民、石材经济 |
| 3 | 中国古代/中世纪 | 青瓦木构、灯笼红 | 宫殿、城墙、市集、宝塔 | 道路规划解锁、税收、士农工商阶层 |
| 4 | 民国/工业革命 | 砖石+蒸汽、雾霾滤镜 | 纺织厂、火车站、钟楼 | 产业链（煤→钢→商品）、铁路交通、污染 |
| 5 | 现代 | 玻璃幕墙、霓虹 | 摩天楼、机场、地铁 | 电力网、 zoning（住宅/商业/工业分区）、汽车交通 |
| 6 | 未来 | 全息蓝紫、垂直城市 | 生态穹顶、磁悬浮、垂直农场 | 清洁能源、自动化、幸福指数取代部分资源 |
| 7 | 太空时期 | 深空黑+星辉、反重力 | 太空电梯、轨道船坞、方舟核心 | 星际资源（小行星采矿）、氧气/生态循环、终局发射 |

**时代跃迁条件**（示例，数值待平衡）：人口阈值 + 关键科技 + 时代奇观 + 幸福度门槛。跃迁时全岛播放 15 秒左右的"时光飞逝"演出：建筑逐个变形升级、昼夜快速轮转、音乐混音过渡。

## 5. 主要玩法系统

### 5.1 建造与规划
- 网格 + 自由摆放混合：道路自由绘制（贝塞尔曲线），建筑吸附网格。
- 建筑分类：住宅 / 生产 / 服务 / 奇观 / 遗迹 / 交通。
- 时代锁定：低时代玩家可预览但无法建造高时代建筑（灰色剪影 + 科技树提示）。

### 5.2 交通系统
- 时代递进：步行小径 → 石板路 → 官道/驿站 → 铁路 → 公路网/地铁 → 磁悬浮 → 反重力航道。
- 简化模拟：不做逐车寻路的重模拟，采用"道路容量 + 热力图"模型，拥堵以车流光带颜色呈现（绿→红），保证性能与观赏性兼得。

### 5.3 人口与市民
- 人口增长 = f(住房, 食物, 幸福度, 医疗)。
- 市民分层渲染：近景个体（有名有姓有表情），远景简化为流动光点/剪影，保证万人级"人潮"性能。
- 幸福度六维：温饱 / 住房 / 工作 / 环境 / 信仰 / 娱乐，雷达图可视化。

### 5.4 经济与资源
- 分时代资源池：食物、木材→石材→煤铁→电力→数据/能源晶体，旧资源不废弃而是降级为"基础物资"。
- 产业链轻量化：每种资源 1~2 级加工即可（如 铁→钢→机器），不做硬核流水线。
- 货币："信仰值"（神力货币）与"金币"（世俗货币）双轨制，互为制约。

### 5.5 神迹与事件
- 祈祷任务（日常）：市民祈求丰收、治病、找工作——完成获信仰。
- 随机事件：海啸、瘟疫、流星雨、商船来访、伟人诞生。
- 神迹列表（消耗信仰）：赐雨 / 沃土 / 治愈 / 启迪 / 流星矿 / 平息怒火 / 时光加速。
- 每条神迹附带"依赖度"隐性数值，滥用会降低市民自主性。

### 5.6 岛屿扩张
- 初始岛约 64×64 格。扩张方式：填海造陆（消耗资源）→ 浮岛（未来时代）→ 轨道平台（太空时代）。
- 地形差异：沙滩/平原/丘陵/矿脉/圣泉，地形决定可建内容，鼓励因地制宜。

## 6. 美术与视听

### 6.1 风格定义：**Low-Poly 三渲二（Cel-Shaded Low-Poly）**
- 低面数模型 + 硬边法线 + 卡通描边（Outline Pass）+ 手绘风色板贴图。
- 色彩策略：每个时代一套主色板（见第 4 节），跃迁时全场景光照 LUT 渐变切换。
- 参考气质：《纪念碑谷》的干净 + 《海岛大亨》的热闹 + 《Cult of the Lamb》的卡通张力。

### 6.2 动画丰富度（重点投入）
- **建筑**：建造时从地面"生长"出来（缩放+尘雾+木架脚手架动画）；升级时逐件替换部件。
- **市民**：走、跑、搬运、欢呼、祈祷、跌倒；节日时成群跳舞。
- **环境**：日夜循环、四季、天气（雨/雪/沙暴/极光）、海面波浪、船只、飞鸟、云层投影。
- **神迹**：全屏粒子演出 + 镜头震动 + 音效压场，让每次施法都有"神的感觉"。
- **镜头**：预设电影机位（时代跃迁、奇观落成自动运镜）。

### 6.3 音频
- 每时代主题 BGM，跃迁时做交叉混音。
- 环境音分层：海浪、市集喧闹、工厂轰鸣、未来城市电子嗡鸣。
- UI 音效用"神圣感"音色（钟声、风铃、低语）强化上帝身份。

## 7. 技术方案

| 层 | 选型 | 理由 |
|---|---|---|
| 语言 | TypeScript | 需求指定 |
| 渲染 | **Three.js** + 自定义 Cel/Outline Shader | Web 3D 生态最成熟，三渲二方案成熟 |
| 框架 | React + Vite | UI 层（HUD/面板/科技树）开发效率高 |
| 状态管理 | Zustand | 游戏状态与 UI 解耦，轻量 |
| 模拟层 | 纯 TS 逻辑模块（ECS 简化版），Web Worker 跑模拟 | 渲染与模拟分离，万人市民不卡主线程 |
| 资源 | glTF + Draco 压缩；程序化生成低模为主 | 低模适合程序化批量生成，控制包体 |
| 存档 | IndexedDB 本地存档 + JSON 导出 | 纯前端可玩，后续可加云存档 |
| 构建 | Vite；目标现代浏览器（WebGL2） | — |

**性能预算**：稳定 60 FPS @ 1080p；同屏建筑 ≤ 3000 实例（InstancedMesh 合批）；市民近景 ≤ 200 个体、远景粒子化。

---

## 8. 游戏模式与边界

- **主模式：自由经营**（无失败压力，沙盒向）。
- **挑战目标（轻量）**：时代跃迁限时挑战、特殊岛屿种子（火山岛/荒漠岛）。
- **MVP 不做**：多人联机、战斗系统（战争仅作为事件文本呈现）、氪金内购。

## 9. 里程碑计划

| 阶段 | 内容 | 交付 |
|---|---|---|
| **M0 原型** | 单岛、远古+古典两时代、建造/人口/资源/信仰核心循环、基础三渲二渲染 | 可玩 Web Demo |
| **M1 纵向切片** | 前 4 个时代（到工业革命）、神迹系统、祈祷任务、交通初版、时代跃迁演出 | 可完整体验"文明进化"核心乐趣 |
| **M2 全时代** | 全部 7 时代 + 终局发射、事件系统、岛屿扩张 | Beta |
| **M3 打磨** | 动画/音频铺满、性能优化、存档、平衡性 | 1.0 |

**建议 M0 范围（确认后即开工）**：
1. Three.js 场景 + 三渲二渲染管线（描边 + 色板 + 日夜光照）
2. 岛屿地形生成 + 格子建造系统（10 种以内建筑）
3. 市民简化 AI（出生→工作→回家，近景个体）
4. 资源三件套（食物/木材/信仰）+ 基础祈祷任务
5. 远古→古典一次完整的时代跃迁演出
6. 镜头：环绕/缩放/跟随 + 基础 UI（资源栏、建造面板、祈祷气泡）
